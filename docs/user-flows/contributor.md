### 1. Dashboard Home

**URL:** `/`

After login, contributors see:

- User menu with profile options
- "Suggest" option visible
- Contribution stats (if available)

### 2. Submit New Suggestion

**URL:** `/suggest`

#### Suggestion Types

**Create New Project:**
Report a construction project not yet in the database.

**Update Existing:**
Propose changes to existing construction data.

**Mark as Completed:**
Report that a project has finished.

**Report Correction:**
Flag incorrect information.

## Notifications

Contributors receive notifications for:

| Event | Notification |
|-------|--------------|
| Suggestion approved | "Your suggestion has been approved!" |
| Suggestion rejected | "Your suggestion was not accepted: [reason]" |
| Changes requested | "Please update your suggestion: [notes]" |
| Suggestion merged | "Your changes are now live!" |

## Best Practices

### Writing Good Suggestions

1. **Be specific** - Clear title and description
2. **Provide evidence** - Link to reliable sources
3. **Accurate location** - Use geometry editor carefully
4. **Complete information** - Fill optional fields when known
5. **One topic per suggestion** - Don't combine multiple updates

### Evidence Guidelines

**Good evidence:**
- Official news sources (TuoiTre, VNExpress)
- Government announcements
- Dated photographs with location
- Official project documents

**Weak evidence:**
- Social media posts
- Undated images
- Personal observations only
- Unverifiable claims

## Reputation System

| Action | Points |
|--------|--------|
| Suggestion approved | +10 |
| Suggestion merged | +5 |
| Suggestion rejected | -2 |

**Reputation Benefits:**
- Higher priority in review queue
- Trust indicators (future feature)
- Potential moderator invitation
